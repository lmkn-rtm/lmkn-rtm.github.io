Sobek Library

Changelog | Latest Release

Overview

Sobek is a parser and API library used for parsing, editing, creating and serializing .sob documents. Purpose of Sobek is to serve as configs and metadata storage file, it's similar to JSON, but a bit more stricter and simpler. Here are the rules defining how Sobek document should look like:

  1. Sobek document should always be encoded in UTF-8
  2. Key names should abide to this regex: [a-zA-Z][a-zA-Z0-9_-]* ; Maximum key lenngth is 255 chars
  3. Object items delimeter is '\n'
  4. Array items delimeter is ',' (can be trailing)
  5. Numbers are double, do not support scientific notation, no hex and no bin
  6. Arrays can store only single type
  7. Root is the only object not surrounded by '{' and '}'
  8. Supported escapes in strings: \n, \t, \", \\
  9. Comments are single line only

With Sobek also come my utils library, needed for compiling. You may use it if you want as well.

Installation

  1. Download

    Get the latest release from GitHub

  2. Extract

    Unzip to your preferred location.

  3. Build

    If you want to recompile the library, open console in src directory and run make.

  4. Link

    Add these flags to your build:

    
                    set SOBEK_PATH="where your libsobek/src is"
                    set SOBEK_FLAGS=-I$(SOBEK_PATH) -L$(SOBEK_PATH) -lsobek -lutils

    -I specifies location of headers (in src there are both .h and .c files as well as Makefile to build it all).

    -L specifies where libsobek.a is - it's in "libsobek/src" folder

    -l flags are used for lib linking, include -lutils too, sobek.h depends on them
