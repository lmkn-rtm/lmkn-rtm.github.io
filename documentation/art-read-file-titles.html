How to read executable file titles

Executable files of the game come with version name and version code in title, giving some information about the type of package one have downloaded.

First three letters tell what type of the game package it is. Let's iterate through them and see what do they mean and what variations may appear:

  1. First letter tells if package contains only data used by game or provided with it's source code available
    1. "[e]" - expanded
    2. "[d]" - data only
  2. Second letter tells if package is snapshot version or release. Snapshots may not have version names:
    1. "[s]" - snapshot
    2. "[r]" - release
  3. Third letter tells version development era:
    1. "[i]" - in development
    2. "[a]" - alpha
    3. "[b]" - beta
    4. "[r]" - release

Following two digits are major version and subversion. Next goes "-AoM-" part and finally version name (May be or may not be).

  1. Example: "arr.3.2-AoM-Name-of-Version.exe"
  2. Explanation: archieved subversion 2 of third release of release era, named "Name of Version".
  3. Detailed explanation: This package has it's data archived, taking less space but not being able to be recompiled. This is a stable release with minimal amount of bugs, tested thoroughly. This package was released during release era, meaning the basic game was already fully released and this is another extension update that adds new feautures.

Fun fact: window title has it's version name written in square brackets like this: "arr.3.2-AoM-[Name of Version]"
